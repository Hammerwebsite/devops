Размер диска?
Использую общеизвестную формулу:
Общее количество данных (Гб) = Сырые данные в день * Количество дней хранения * фактор трансформации данных (условно 1) * (количество реплик + 1)
23 * 60  * 2  = 2760Гб 
  
Общее хранилище данных (Гб) = Общее количество данных (Гб) *(1 + 0,15 запаса + 0,05 дополнительные нужды)
2760 * (1+ 0,15*2760 +  0,05*2760) = 3312

Общее количество нод = Общее хранилище данных (Гб) / Объём памяти на ноду / соотношение памяти к данных)
3312 / 20 / 60 = 2,58 ~ 3шт
0,010416
Использую storage ratio 1:60 рекомендуется для хранения и анализа логов

Расскажите, почему выбрали такую конфигурацию.


Нагрузка по ТЗ не большая. Требуется минимальный кластер для обеспечения репликации и отказоустойчивости. Для каждого шарда будет одна реплика. Для корректной работы алгоритма голосования (кворума) в случае выхода из строя одной из нод а так-же обеспечения отказоустойчивости требуется минимум три ноды. 

Под JVM Heap не более 50% общего объёма памяти и не более 30Гб во избежание набега garbage collector. Оставшаяся память будет использована как кэш операционной системы.
